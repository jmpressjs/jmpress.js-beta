Route

By default, jmpress.js will flow through each step linearly down the DOM. You have a couple of ways to modify the flow.

Inline Next / Prev


	
A

	
B

	
C

	
D

The flow will start with A, goto C, then D and back to A. Only when the user is on C and goes back will they be shown B.

Using route:

You can set the flow using the route method. You also don't have to define all the steps. The example below will preserve the natural flow and only modify the indicated steps:

$("#jmpress").jmpress("route", ["#b", "#d", "#c"]);

The flow will start with A, goto B, goto D, goto C and back to A.

Unidirectional:

$("#jmpress").jmpress("route", ["#d", "#b"], true);

The flow will be A -> B -> C -> D -> B -> C -> D -> A.
